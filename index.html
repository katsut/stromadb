The knowledge graph your agents can query in real time.

StromaDB fuses vectors × types × time in one engine: a streaming knowledge graph where an LLM agent runs type-aware hybrid search, reads any predicate as-of a point in time, and gets an answer with its provenance — while the graph keeps ingesting underneath.

Capabilities

One engine for the way agents actually read a graph.

Retrieval that is semantic, structural, and temporal at once — not a vector store bolted to a graph db.

Type-aware hybrid search

Vector similarity scoped by node type and graph structure, authz-filtered at the source — so an approximate ANN × a type filter doesn't collapse recall. Types are declared in a lightweight ontology (domain/range, cardinality, relation properties) — no axioms, no reasoner.

filtered recall@10 ~1.0 vs ~0.13 plain-ANN

Valid-time, as-of reads

Every fact carries a validity interval. Ask for the value in effect at any instant — "who approved this, as of the approval time" — not just the latest write.

bitemporal · point-in-time queries

Real-time under writes

Lock-free reads over a pinned snapshot: a long streaming ingest never stalls a reader, and each read is snapshot-isolated.

read p99 1.32ms under a concurrent writer

Declared rules → verdicts

Evaluate a declared rule over the graph deterministically: what's expected-but-absent, and what's present-but-wrong (with an as-of derived path). No reasoner.

conformance & completeness ops

Durable by design

Framed write-ahead log with group-commit fsync; cold-start replays the committed prefix and drops a torn tail via checksum.

RTO 0.81s @5M facts · 0 data loss

Agent-native (MCP)

Ships a Model Context Protocol server: an LLM discovers the schema and calls the graph as tools — schema, point, expand, search, conformance — over stdio.

works with Claude, GPT, any MCP client

For agents

Hand the deterministic part to the engine.

An LLM is great at orchestration and language, unreliable at exact multi-hop, as-of graph logic. So the engine evaluates declared rules and returns a deterministic verdict; the agent calls it and acts.

declare a rule once
// approver = manager of the assignee's dept,
// as of the approval time
{ "op": "conformance",
  "rule": {
    "subject_type": "Issue",
    "required": { "hops": [
      {"predicate":"assigned-to"},
      {"predicate":"member-of"},
      {"predicate":"manager-of",
       "as_of":"approved-at"} ] },
    "actual": "approved-by" } }
get a verdict per subject
{ "verdicts": [
  { "subject":1001, "verdict":"OK" },
  { "subject":1004, "verdict":"MISMATCH",
    "kind":"wrong" },
  { "subject":1005, "verdict":"MISMATCH",
    "kind":"stale",     // approved by a
    "required":{"node":12}, // past manager
    "actual":{"node":10}, "as_of":6000 }
] }
